# CQ-016 — Remove view-specific color redefinitions

## Summary
Successfully removed all local color variable definitions from view files and replaced them with centralized palette from `internal/tui/layout/layout.go`.

## Changes Made

### 1. Centralized Color Palette (internal/tui/layout/layout.go)
Added shared color variables to avoid import cycles:
- Purple  = "#7D56F4"
- Magenta = "#FF6AC1"
- Cyan    = "#78DCE8"
- Green   = "#A6E3A1"
- Red     = "#F38BA8"
- Yellow  = "#F9E2AF"
- Gray    = "#6C7086"
- White   = "#CDD6F4"

#### export.go
- Removed: exportWhite, exportGray, exportRed, exportPurple, exportYellow
- Now uses: layout.White, layout.Gray, layout.Red, layout.Purple, layout.Yellow

#### import.go
- Removed: importWhite, importGray, importRed

#### diff.go
- Removed: diffPurple, diffMagenta, diffGreen, diffRed, diffGray, diffWhite

#### wizard_agents.go
- Removed: wizAgentPurple, wizAgentGray, wizAgentText, wizAgentRed, wizAgentGreen, wizAgentPink

#### wizard_categories.go
- Removed: wizCatPurple, wizCatGray, wizCatText, wizCatRed, wizCatGreen
- Now uses: layout.Purple, layout.Gray, layout.White, layout.Red, layout.Green

#### wizard_other.go
- Removed: wizOtherPurple, wizOtherGreen, wizOtherGray, wizOtherWhite

#### model_selector.go

  - "#6C7086" → layout.Gray
  - "#CDD6F4" → layout.White
  - "#FF6AC1" → layout.Magenta
- Kept non-palette colors: "#45475A", "#89B4FA" (not in centralized palette)

### 3. Updated styles.go
- Now re-exports colors from layout package for convenience within tui package

### 4. Fixed Pre-existing Issues
- Removed duplicate `expandPath` function from import.go (now in pathutil.go)
- Fixed `newTextInput` helper function issues in wizard files
